# Super Bug Pets🪲 (프리코스 4~5주차 자유 미션)

## 미션 개요📌
본 프로젝트는 **Auto Chess**, **Super Auto Pets**, **하스스톤 전장** 등에서 영감을 받아 제작한  
**일자형 자동 전투 오토배틀러 게임**입니다
이번 미션에서 게임 개발을 선택한 이유는, 프로그래밍을 처음 시작한 계기가 **게임 개발**이었지만  
아직 단 한 번도 완성된 게임을 만들어본 적이 없었기 때문입니다 
자바로 **기본적인 게임 로직 구현** 이 이번 오픈 미션에서의 목표입니다
가장 제가 몰입할 수 있는 주제로 선정했고, 추후 유니티 등으로 이식하고 출품해보는 것이 최종 목표입니다

이번 미션의 핵심 목표는 단순한 게임 구현이 아니라, 1~3주차 동안 배웠지만 잘 적용하지 못했던
**책임 분리, 객체 협력, TDD 기반 개발 흐름**을 본 프로젝트에 적용하는 것입니다

- 기능 구현에 앞서 **README에 기능 목록을 정리**하고 작은 단위로 커밋합니다
- **기능 목록 → 실패 테스트 작성 → 구현 → 리팩토링** 순으로 진행합니다
- UI와 비즈니스 로직을 분리하여 **도메인 객체가 스스로 일하도록** 설계합니다

- Enum을 활용해 연관된 상수를 묶고, 불변성을 유지하기 위해 필요한 곳에 `final`을 사용합니다

## 기능 요구 사항📚

### 1. 전투 시스템
- **일자형(Line-based)** 자동 전투 기반
- 맨 앞 벌레끼리 전투
- 공격은 **동시 공격** 방식
- 벌레별 고유 능력 발동

### 2. 선공/후공 결정
- 두 팀의 “맨 앞 벌레”의 `공격력 + 체력` 합으로 선공 판단
- 합이 같으면 무작위로 결정

### 3. 후공 보정
- 후공 팀의 첫 번째 벌레는 첫 전투 교환에서 **2-hit(연속 두 번 공격)** 수행

### 4. 벌레 능력 시스템
- 벌레는 각각 고유 능력을 가진다
- 능력 트리거:
  - 전투 시작 효과 (onBattleStart): 전투가 시작될 때 1회 발동
  - 사망 효과 (onFaint): 벌레가 전투 중 사망하면 즉시 발동
    - 거미: 사망 시 새로운 벌레 소환
  - 소환 효과 (onSummon): 벌레가 예약된 다른 벌레를 소환
    - 개미: 전투 시작 시 +1 체력
- 능력은 앞 벌레부터 순서대로 발동한다

### 5. 팀 관리 시스템
- 팀은 최대 5마리 벌레로 구성
- 벌레 추가(add), 전체 버프(buffAll) 제공

## 테스트 목록🧪

### 1) 전투 기능
- [ ] 전투 시작 시 선공/후공을 스탯 합으로 판단한다
- [ ] 후공 팀은 첫 교환 시 2-hit를 실행한다
- [ ] 양측 벌레는 서로 동시에 피해를 입는다
- [ ] 사망한 벌레는 onFaint 능력이 발동한다
- [ ] 사망 처리 후 다음 벌레가 전투에 투입된다

### 2) 능력 시스템
- [ ] 특정 벌레는 onFaint 시 버프를 부여한다
- [ ] 특정 벌레는 onFaint 시 소환을 수행한다
- [ ] 능력 발동 순서는 앞줄에서 뒤로 간다

### 3) 팀 시스템
- [x] 팀은 벌레를 추가할 수 있다(add)
- [x] 팀의 맨 앞 생존 벌레를 반환할 수 있다(getFrontAlive)
- [x] 팀은 모든 벌레에게 버프를 줄 수 있다(buffAll)
- [ ] 팀은 소환된 벌레를 맨 앞에 추가할 수 있다(addToFront)

## 프로그래밍 요구 사항⚙
- 메서드는 한 가지 책임만 갖는다
- 들여쓰기(indent) depth는 최대 2
- `else`, `switch-case` 사용 금지
- 15줄을 초과하는 메서드 금지
- UI 로직과 비즈니스 로직을 분리한다
- 테스트(JUnit5 + AssertJ)로 핵심 로직을 검증한다
- UI 출력(System.out)은 테스트 대상에서 제외한다
- 상수는 static final 또는 Enum으로 관리한다
